# Play Console Web Submission Checklist

Use this file once you already have the signed `.aab`.

Date baseline: February 16, 2026.

## Create app record (first release only)

- [ ] Open Google Play Console
- [ ] Create app
- [ ] App name: `Oh Craps!`
- [ ] Default language: choose your primary language
- [ ] App or game: `App`
- [ ] Free or paid: choose business model
- [ ] Package name must match upload artifact: `com.marconius.ohcraps`

## Complete store listing

- [ ] Fill App details
- [ ] Fill short description
- [ ] Fill full description
- [ ] Upload icon, feature graphic, screenshots
- [ ] Add contact details
- [ ] Add privacy policy URL (your existing hosted policy)

Use the prepared file:

- `05-store-listing-fields-and-assets.md`

## Complete App content and policy declarations

- [ ] Privacy policy
- [ ] Ads declaration
- [ ] App access
- [ ] Content rating questionnaire
- [ ] Target audience and content
- [ ] News apps declaration (if applicable)
- [ ] Data safety

Use the prepared file:

- `03-policy-and-declarations-workbook.md`

## Create production release

- [ ] Go to Production
- [ ] Create new release
- [ ] Upload `app-release.aab`
- [ ] Review automatically generated artifacts
- [ ] Paste release notes from `04-release-notes-templates.md`
- [ ] Save release

## Final checks before rollout

- [ ] Resolve policy warnings, blockers, and missing tasks
- [ ] Confirm no unresolved pre-launch report critical issues
- [ ] Confirm Data safety answers match current app behavior
- [ ] Confirm privacy policy URL loads publicly

## Start rollout

- [ ] Start production rollout
- [ ] Monitor status until review completes
- [ ] Track approval, rejection, or requested changes

## Post-release

- [ ] Verify listing is live in Play Store
- [ ] Install live build from Play Store and smoke test
- [ ] Archive release notes and artifact metadata for next cycle
